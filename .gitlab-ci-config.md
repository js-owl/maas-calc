# GitLab CI/CD Configuration

This document explains the GitLab CI/CD pipeline configuration for the Manufacturing Calculation API.

## Pipeline Overview

The pipeline consists of 2 main stages:

1. **Build** - Docker image creation and registry push
2. **Deploy** - Production deployment via SSH

## Stages Breakdown

### Build Stage

#### Docker Build (`build`)
- Builds Docker image using multistage Dockerfile.prod
- Uses BuildKit for faster builds and caching
- Tags image with commit SHA and 'latest'
- Pushes to private Docker registry
- Only runs on Git tags

### Deploy Stage

#### Production Deployment (`deploy`)
- Deploys to production environment via SSH
- Uses docker-compose.prod.yml with Traefik integration
- Includes health verification
- Only runs on Git tags
- Depends on successful build

## Required GitLab Variables

Set these variables in GitLab Project Settings > CI/CD > Variables:

```
DOCKER_USERNAME - Username for Docker registry
DOCKER_PASSWORD - Password for Docker registry (mark as masked)
```

## Environment URLs

- **Staging**: https://staging-api.your-domain.com
- **Production**: https://api.your-domain.com

## Docker Registry

The pipeline uses a private Docker registry at `vortex.kronshtadt.ru:8443/maas-proxy`.

## Branch Strategy

- **tags**: Production releases, triggers full build and deployment
- **merge_requests**: No automatic pipeline (can be enabled for testing)
- **feature/***: Development branches, no automatic pipeline

## Artifacts

The pipeline generates:

- Docker images in private registry
- Deployment logs on remote server

## Monitoring and Notifications

- Pipeline status is visible in GitLab UI
- Deployment verification includes health checks
- Deployment logs are created on remote server

## Future Enhancements

Additional pipeline stages can be added as needed. See `.gitlab-ci-future.yml` for examples of:

### Test Stages
- **Unit Tests**: Automated pytest execution with coverage
- **Integration Tests**: End-to-end API testing
- **Code Quality**: Linting, formatting, and type checking
- **Performance Tests**: Load testing with Locust

### Security Stages
- **Trivy Scanning**: Docker image vulnerability scanning
- **Bandit Scanning**: Python code security analysis
- **Dependency Scanning**: Package vulnerability checks

### Additional Features
- **Staging Deployment**: Automatic staging environment updates
- **Notifications**: Slack/email alerts on pipeline events
- **Multi-environment**: Support for multiple deployment targets

## Customization

To customize this pipeline for your environment:

1. Update registry URLs and image names in variables
2. Modify deployment scripts for your infrastructure
3. Add additional stages from `.gitlab-ci-future.yml` as needed
4. Configure notification channels

## Security Considerations

- Docker credentials are stored as masked variables
- SSH keys are stored as masked variables
- Production deployments are automated but can be made manual
- Old images can be cleaned up manually
